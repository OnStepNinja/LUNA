## 2. コネクタ・ケーブル仕様

### Mini-DIN 4ピン ピンアサイン（マウント側・オスコネクタ正面図）

```
[3] [4]
[1] [2]
```

| ピン | 信号 | 方向 | 備考 |
|------|------|------|------|
| 1 | TXD | マウント → PC | マウント送信 |
| 2 | RXD | PC → マウント | マウント受信 |
| 3 | GND | — | グランド |
| 4 | 制御信号 | — | 通常 NC |

> **出典**: Takahashi 公式ケーブル図（ASCOM Temma Driver PDF p.34）
> Mini-DIN 4P Male → D-sub 9P Female のクロス接続

### Takahashi 純正ケーブル 配線（ASCOM Driver PDF 原文より）

```
Mini-DIN 4P (Male)          D-sub 9P (Female)
     ピン1  ────────────────  ピン2  (RXD: PC受信 = マウント送信)
     ピン2  ────────────────  ピン3  (TXD: PC送信 = マウント受信)
     ピン3  ────────────────  ピン5  (GND)
     ピン4  ────────────────  ピン6  (DSR)
                              ピン7  ─┐
                              ピン8  ─┘ (RTS-CTS ループ)
```

### ESP32 接続（AiBridgeMCP v2.1.0 設計）

```
Temma2 Mini-DIN4     MAX3232 モジュール      ESP32
  ピン3 (GND) ──────── GND ─────────────── GND
  ピン1 (TXD) ──────── R1IN → R1OUT ────── GPIO16 (RX2)
  ピン2 (RXD) ──────── T1IN ← T1OUT ────── GPIO17 (TX2)
  ピン4 (NC)  ─ 未接続
```

> ⚠️ MAX3232 は RS-232C (±12V) ↔ TTL 3.3V 変換に必須
> ⚠️ ピン番号はコネクタ表記方向によって記述が異なる場合があります。
>    実機接続時は TXD/RXD の導通を必ず確認してください。

## 3. 座標フォーマット

**出典**: ASCOM Temma V2 Driver Troubleshooting 節（原文）
> *"The Temma protocol reports coordinates to the nearest second in right ascension (15 arcseconds at 0 degrees declination) and to the nearest 6 arcseconds in declination."*
> *"The Temma's control system accepts declination coordinates to the nearest 12 arcseconds."*

これにより座標は **ASCII 10進数形式** であることが確定します。

### 赤経 (RA) フォーマット：`HHMMSSs`（7桁）

| フィールド | 桁数 | 内容 | 例 |
|-----------|------|------|-----|
| HH | 2 | 時（00〜23） | `10` |
| MM | 2 | 分（00〜59） | `30` |
| SSs | 3 | 秒×10（000〜599） | `456` = 45.6秒 |

例: `1030456` = 10h 30m 45.6s

### 赤緯 (Dec) フォーマット：`SDDMMSS`（7桁）

| フィールド | 桁数 | 内容 | 例 |
|-----------|------|------|-----|
| S | 1 | 符号（`+` または `-`） | `-` |
| DD | 2 | 度（00〜90） | `32` |
| MM | 2 | 分（00〜59） | `10` |
| SS | 2 | 秒（00〜59） | `30` |

例: `-321030` = −32° 10′ 30″

## 4. コマンドリファレンス

> ⚠️ 以下のコマンドは調査・実装済みですが、実機テストによる最終確認が必要です。
> 全コマンドの末尾に **CR (0x0D)** を付加してください。

### 4.1 現在位置取得

```
送信: Q\r
応答: <HHMMSSs><SDDMMSS>  (14文字 + 終端)
```

例:
```
送信: Q
応答: 1030456-321030
      ↑      ↑
      RA     Dec
```

### 4.2 自動導入（Goto）

```
送信: G<HHMMSSs><SDDMMSS>\r
応答: >  (Ready プロンプト、または応答なし)
```

例:
```
送信: G1030456-321030\r
       = RA 10h30m45.6s、Dec -32°10'30" へ導入
```

### 4.3 初期化コマンド群

起動後、最初に導入を行う前に以下の順序で初期化が必要です。

```
1. 緯度設定:  I+<DDMMSS>\r  または  I-<DDMMSS>\r
              例) I+354600\r = 北緯 35°46'00"

2. 時刻設定:  T<HHMMSS>\r
              例) T203045\r = 20:30:45 (JST)

3. 設定確定:  E\r
```

> ⚠️ 各コマンドの正確な応答形式は実機テストで確認が必要です。

## 5. 動作上の注意事項

### Computer Standby スイッチ（重要）

Temma2 本体の **Computer Standby スイッチを ON** にしないと RS-232C 通信が無効になります。
このスイッチが OFF の場合、コマンドを送信しても応答がありません。

### 電源投入順序

起動時の文字化け対策として、以下の順序を推奨します：

```
1. ESP32 (AiBridgeMCP) 起動 → WiFi 接続完了を確認
2. Temma2 電源 ON
```

ESP32 起動時のウェルカムメッセージが Temma2 に送信される問題への暫定対策です。

### プロトコルの一貫性

複数の独立したドライバ開発者（ASCOM・INDI）が確認：
**「Temma PC と Temma2 でプロトコルは変更なし」**
（同一ドライバ・同一ケーブルで動作）

| 項目 | 精度 | 備考 |

## 7. 実機テストで確認すべき事項

実機（Temma2）接続後、以下を優先して確認してください：

| 優先度 | 確認内容 | 確認方法 |
|--------|---------|---------|
| **最高** | `Q\r` への応答有無 | `serial_query("Q\r", 1000)` を送信 |
| **最高** | Q コマンド応答フォーマット | レスポンス文字列を記録 |
| **最高** | 19200bps で通信可能か | 接続確立の可否 |
| **高** | `G` コマンドへの応答（`>`か否か） | 導入テスト |
| **高** | 初期化なし導入の可否 | I/T/E なしで G コマンド |
| 中 | ピン番号（TXD/RXD）の正確な対応 | TXD↔RXD 入れ替えテスト |

## 8. 不採用プロトコル（記録）

調査中に提示されたが、**裏付けが取れず採用しなかったプロトコル仕様**：

- `X<hhhhhhh>` / `Y<hhhhhhh>` / `S` による 3 ステップ導入
- 24 ビット 16 進数座標（7 桁 hex）
- `Z` ステータス（ビットフラグ）/ `H` ハード停止

上記コマンドは公開されている ASCOM/INDI ドライバの一次資料に**一切記載がなく**、採用しないこととします。
